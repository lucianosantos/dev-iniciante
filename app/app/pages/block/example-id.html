1) IA (modo Estudo) — como aprender melhor

Overview

Ative o modo Estudar e aprender do ChatGPT. A proposta é um tutor que pergunta antes de responder, valida raciocínio e sugere exercícios graduais. Aqui você configura seus prompts-base para usar durante todo o curso.

Vídeos (PT-BR)

  • Testando o Modo Estudo do ChatGPT (demonstração)
  • Como usar ChatGPT para estudar programação (exemplos práticos)
  • 8 formas de estudar com ChatGPT (resumos, quiz, mapas mentais)

Artigos (PT/pt‑BR)

  • OpenAI (pt‑BR): Apresentamos o modo Estudo
  • DIO: Guia de Engenharia de Prompts

Cheat sheet — prompts úteis

  • Diagnóstico: “Pergunte meu nível sobre Git; 5 questões; explique meus erros e recomende 1 exercício.”
  • Socrático: “Não entregue a resposta. Faça perguntas curtas e confirme meu raciocínio a cada passo.”
  • Revisão de commit/PR: “Avalie a clareza e escopo segundo Conventional Commits; sugira melhor mensagem.”
  • Comparativo: “Explique reset --soft × --mixed × --hard com exemplos.”
  • Flashcards: “Crie 10 flashcards em PT‑BR sobre Git; aplique um quiz aleatório.”
  • Registro: “Resuma em 5 bullets o que aprendi hoje + 1 próximo passo prático.”
Dica: salve esses prompts em docs/prompts-ia.md no seu repositório.

2) Git (fundamentos) — do zero à colaboração

Overview

Você sai do zero, entende conceitos (repositório, staging, commit, branch, remoto, PR) e pratica o fluxo status → add → commit → pull --rebase → push. Adota mensagens padronizadas e PR curto.

Vídeos (sequência sugerida)

  • Rafaella Ballerini — Parte 1: conceitos (iniciante)
  • Rafaella Ballerini — Parte 2: prática (init, add, commit, PR)
  • Minicurso Git em ~45 min (revisão prática)
  • Vídeo único (guia completo) — revisão final

Leitura

  • Conventional Commits (pt‑BR)
  • Mensagens de commit: guia em português
  • Pro Git (pt‑BR) — capítulo básico

Cheat sheet

  • GitHub — Folha de Dicas (pt‑BR, PDF)

Exercícios

  • Configurar global (user.name, user.email, init.defaultBranch=main)
  • Repo local → primeiro commit → push no GitHub
  • Criar feat/onboarding, editar README e abrir 1 PR
  • Simular conflito simples e resolvê‑lo; usar git log --oneline --graph
  • Reescrever 3 mensagens usando Conventional Commits

3) Projeto / Entregável — repo do bloco

Estrutura sugerida do repositório

bloco-01-git/
├─ README.md                  # objetivos, como rodar, convenção de commits
├─ LICENSE
├─ CHANGELOG.md
├─ docs/
│  ├─ diario-erros.md         # o que quebrei / como resolvi (semanal)
│  └─ prompts-ia.md           # prompts úteis de estudo e revisão
└─ .github/
   ├─ PULL_REQUEST_TEMPLATE.md
   └─ ISSUE_TEMPLATE.md

Checklist de entrega

  • [ ] Git configurado e primeiro push
  • [ ] Branch strategy descrita no README
  • [ ] Templates .github/ ativos (PR/Issue)
  • [ ] 1 PR revisado (humano + IA) e mergeado
  • [ ] CHANGELOG iniciado
  • [ ] Diário de erros criado e preenchido
  • [ ] Prompts de IA salvos e testados

Resumo rápido

IA primeiro (modo Estudo) → depois Git. Use a IA para diagnóstico, revisão e revisão espaçada. O entregável é um repo padronizado com 1 PR real.

Tempo sugerido (~40h)

  • IA (modo estudo): 8–10h
  • Git fundamentos: 18–20h
  • Projeto/PR + documentação: 10–12h
